# ndm-test

* При помощи docker compose поднимается 4 контейнера (app с js приложением, nginx(порт 8080), nginx1(порт 8081), nginx2(порт 8082))
* При обращении к прокси без дополнительного пути запрос будет передаваться на app
* При обращении к прокси c путем /proxy/ запрос будет передаваться по цепочке nginx2 > nginx1 > nginx, а потом на app

  Протокол тестирования:

  ```
C:\Users\user>curl -H "X-Forwarded-For: 203.0.113.195, 70.41.3.18" http://81.26.190.115:8080  
{"X-Forwarded-For":"109.124.116.71, 172.18.0.3"}  
C:\Users\user>curl -H "X-Forwarded-For: 203.0.113.195, 70.41.3.18" http://81.26.190.115:8081  
{"X-Forwarded-For":"109.124.116.71, 172.18.0.4"}  
C:\Users\user>curl -H "X-Forwarded-For: 203.0.113.195, 70.41.3.18" http://81.26.190.115:8081/proxy/  
{"X-Forwarded-For":"109.124.116.71, 172.18.0.4, 172.18.0.3"}  
C:\Users\user>curl -H "X-Forwarded-For: 203.0.113.195, 70.41.3.18" http://81.26.190.115:8082  
{"X-Forwarded-For":"109.124.116.71, 172.18.0.5"}
  ```
